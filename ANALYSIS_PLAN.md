## 3. 분석 단위 결정: 전체 학습 후 구간별 집계

### 3.1 핵심 질문

> K-Means와 LDA를 **4개 구간으로 나눠서** 각각 돌릴 것인가, 아니면 **전체 8,972건을 합쳐서** 한 번 돌릴 것인가?

### 3.2 구간별 분리 학습의 문제점

구간별로 나눠서 모델을 따로 학습하면 **각 구간이 독립된 토픽/군집 공간**을 가지게 된다.

예를 들어 K-Means(k=3)를 구간마다 따로 돌리면:

```
구간 1의 Cluster 0 = "소아과 붕괴 우려" 담론
구간 2의 Cluster 0 = "전공의 집단 사직" 담론   ← 완전히 다른 주제
```

두 Cluster 0은 같은 ID를 공유하지만 전혀 다른 의미를 가진다.
즉, **"구간 1의 Cluster 0이 구간 2에서 줄었다"는 비교 자체가 성립하지 않는다.**
시계열 변화를 보려고 구간을 나눴는데, 정작 구간 간 비교가 불가능해지는 모순이 생긴다.

LDA도 동일하다. 구간마다 따로 학습하면 "구간 1의 Topic 2"와 "구간 2의 Topic 2"가 서로 다른 주제를 가리키므로, 토픽이 시간에 따라 부상하거나 소멸했는지 추적할 수 없다.

### 3.3 전체 합산 후 구간별 집계 방식 (채택)

**전체 8,972건을 하나의 말뭉치로 학습**하면 모든 문서가 동일한 토픽/군집 공간을 공유한다.

```
[학습] 전체 8,972건 → K-Means / LDA 단일 모델 학습
         ↓
[할당] 각 문서에 Cluster ID / dominant Topic ID 부여
         ↓
[집계] section 컬럼으로 그룹핑
         ↓
[비교] 구간별 각 군집·토픽의 문서 비율 변화 시각화
```

이렇게 하면 "Cluster 0(의료 현장 우려)"이 구간 1에서는 40%, 구간 4에서는 15%처럼 **같은 레이블로 시계열 변화를 직접 추적**할 수 있다.

### 3.4 보완책: 구간별 미묘한 담론은 TF-IDF로 커버

전체 합산 시 특정 구간에서만 나타난 미묘한 담론이 묻힐 수 있다는 한계가 있다.
이는 이미 완료한 **구간별 TF-IDF 상위 키워드** 분석으로 보완한다.

| 분석                      | 역할                                        |
| ------------------------- | ------------------------------------------- |
| K-Means / LDA (전체 학습) | 전체 담론 구조 파악 + 구간별 비율 변화 추적 |
| TF-IDF (구간별)           | 구간 고유의 키워드 변화 포착                |

## 4. K-Means 군집 분석

### 4.1 방법론

| 항목          | 설정                                          |
| ------------- | --------------------------------------------- |
| 알고리즘      | KMeans                                        |
| 군집 수(k)    | **3**                                         |
| 벡터화        | `TfidfVectorizer(min_df=5, max_df=0.85)`      |
| 초기화        | `n_init=20, random_state=42`                  |
| **분석 단위** | **전체 8,972건 단일 학습** → 구간별 분포 집계 |
| 특징 수       | 전체 어휘 기준 (64,815개)                     |

**k=3 선택 근거**: 엘보우 분석(SSE vs K) 결과, k=3에서 SSE 감소율이 뚜렷이 꺾이는 지점이 확인됨.

### 4.2 분석 절차

```
1. 전체 doc_text_filtered (8,972건) 로딩
2. TF-IDF 행렬 생성 (전체 말뭉치 기준 희귀어·과다등장어 제거)
3. KMeans(k=3) 단일 학습 → 각 문서에 Cluster ID 할당
4. 군집 중심(centroid) 기준 상위 15개 키워드 추출 → 담론 유형 레이블링
5. section 컬럼으로 그룹핑 → 구간별 군집 비율 계산
6. 구간별 군집 비율 변화 시각화 (스택 막대 / 선 그래프)
```

### 4.3 출력 파일

| 파일                                                      | 내용                             |
| --------------------------------------------------------- | -------------------------------- |
| `outputs/pipeline/kmeans/kmeans_cluster_top_terms.csv`    | 군집별 상위 키워드 + 가중치      |
| `outputs/pipeline/kmeans/kmeans_doc_assignments.csv`      | 문서별 Cluster ID + section      |
| `outputs/pipeline/kmeans/kmeans_section_distribution.csv` | 구간별 군집 비율 (시계열 추적용) |
| `outputs/pipeline/kmeans/kmeans_elbow.csv`                | 엘보우 분석 SSE 수치             |

### 4.4 담론 유형 레이블링 계획

전체 학습 후 3개 군집의 centroid 키워드를 보고 **담론 유형** 레이블을 부여한다.

| 레이블 후보      | 대표 키워드 경향                             |
| ---------------- | -------------------------------------------- |
| 의료 현장 우려형 | 붕괴, 현장, 간호사, 근무, 환자, 소아청소년과 |
| 정책·교육 논쟁형 | 정원, 교육, 입학, 정부, 개혁, 법원           |
| 사회·경제 파급형 | 투자, 미래, 경쟁, 시스템, 비용, 반도체       |

레이블링 후 구간별 비율 변화를 통해 아래를 확인한다:

- 초기(구간 1)에는 어떤 담론이 지배적이었는가
- 사태 장기화(구간 3)에서 담론 분포가 어떻게 이동했는가
- 구간 4(복귀 국면)에서 새로운 담론이 부상했는가

## 5. LDA 토픽 모델링

### 5.1 방법론

| 항목          | 설정                                          |
| ------------- | --------------------------------------------- |
| 알고리즘      | `LatentDirichletAllocation` (sklearn)         |
| 토픽 수       | **5개**                                       |
| 벡터화        | `CountVectorizer(min_df=5, max_df=0.85)`      |
| 학습 방식     | `learning_method='batch', random_state=42`    |
| **분석 단위** | **전체 8,972건 단일 학습** → 구간별 분포 집계 |
| 키워드 수     | 토픽당 상위 15개                              |

### 5.2 LDA 수행 과정

```
1. 문서단어행렬(DTM) 생성
   - CountVectorizer로 전체 8,972건 벡터화
   - min_df=5: 5건 미만 등장 어휘 제외
   - max_df=0.85: 85% 이상 문서 등장 어휘 제외 (너무 흔한 단어)

2. 토픽 수 5 사전 설정
토픽 1 = topic_0
대표어: 재생, 마음, 엄마, 부모, 친구, 경험, 직업, 생활, 고민
해석: 일상/경험/부모-자녀 서사형
메모: 의대 증원 핵심 담론이라기보다 잡음이나 주변 생활형 문서가 섞인 느낌이 큽니다.

토픽 2 = topic_1
대표어: 재수, 점수, 학습, 설명회, 국어, 약대, 학종, 연세대, 원서
해석: 입시/학습/학부모 관심형
메모: 의대 증원이 입시에 미치는 영향 쪽 담론으로 보입니다.

토픽 3 = topic_2
대표어: 사직, 위원회, 단체, 현장, 제출, 기관, 복귀, 응급, 대책, 갈등
해석: 전공의 사직/복귀/의료현장 갈등형
메모: 이게 가장 전형적인 의대 증원 본류 담론 중 하나예요.

토픽 4 = topic_3
대표어: 수가, 지지, 주장, 민주당, 정권, 선거, 인구, 전문의, 민영, 시스템
해석: 정치/정책/수가 논쟁형
메모: 정책 책임, 정치 프레임, 의료제도 논쟁이 섞인 토픽으로 보입니다.

토픽 5 = topic_4
대표어: 반도체, 투자, 중국, 금리, 사업, 국내, 주식, 기술, 전망, 부동산
해석: 경제/투자/산업 잡음형
메모: 이건 의대 증원과 직접 관련 없는 문서가 꽤 섞였다는 신호로 보입니다.

3. 모든 단어 → 5개 토픽 중 하나 무작위 초기 할당

4. 반복 최적화 (Batch EM)
   - 각 단어 제외 후 재할당:
     - 각 문서 내 토픽 분포 (θ) 계산
     - 각 토픽 내 단어 분포 (φ) 계산
   - 수렴까지 반복

5. 최종 출력
   - 각 문서의 dominant topic ID
   - 각 토픽의 상위 15개 키워드 + 가중치
   - section 컬럼으로 그룹핑 → 구간별 토픽 비율 집계
```

### 5.3 출력 파일

| 파일                                                  | 내용                               |
| ----------------------------------------------------- | ---------------------------------- |
| `outputs/pipeline/lda/lda_topic_keywords.csv`         | 토픽 ID·순위·키워드·가중치         |

## 6. 시각화 계획

### 6.1 히트맵 (Heatmap)

히트맵은 구간 × 토픽/군집 비율을 행렬로 표현하여 **담론 지형의 변화를 한눈에** 파악하게 한다.
수치 테이블로는 놓치기 쉬운 패턴(특정 토픽이 특정 구간에서 급증·급감)을 시각적으로 즉시 포착할 수 있다.

| 히트맵 종류          | 행                  | 열       | 값                  | 파악 가능한 것                           |
| -------------------- | ------------------- | -------- | ------------------- | ---------------------------------------- |
| LDA 토픽 × 구간      | 토픽 0~4            | 구간 1~4 | 해당 토픽 문서 비율 | 어떤 토픽이 어느 구간에서 지배적이었는가 |
| K-Means 군집 × 구간  | 군집 0~2            | 구간 1~4 | 해당 군집 문서 비율 | 담론 유형별 구간별 세력 변화             |
| TF-IDF 키워드 × 구간 | 상위 키워드 20~30개 | 구간 1~4 | TF-IDF 가중치       | 키워드 중요도의 시계열 증감              |

**구현 방법**: `seaborn.heatmap()` + `annot=True`로 수치 병기, 컬러맵은 `YlOrRd` (낮음=흰색, 높음=붉은색)

```python
# 예시: LDA 토픽 × 구간 히트맵
pivot = doc_df.groupby(['section', 'dominant_topic']).size()
pivot = pivot.unstack().div(pivot.unstack().sum(axis=1), axis=0)  # 구간별 비율
sns.heatmap(pivot.T, annot=True, fmt='.2f', cmap='YlOrRd')
```

### 6.2 박스플롯 (Boxplot)

박스플롯은 군집·구간별로 **반응 지표(좋아요·댓글)의 분포**를 보여준다.
단순 평균이 아닌 분포 형태(중앙값, 이상치, 산포도)를 통해 담론 유형 간 여론 반응 특성의 차이를 드러낸다.

| 박스플롯 종류       | X축            | Y축         | 파악 가능한 것                        |
| ------------------- | -------------- | ----------- | ------------------------------------- |
| 군집별 좋아요 분포  | Cluster ID     | like 수     | 어떤 담론 유형이 더 공감을 받는가     |
| 군집별 댓글 수 분포 | Cluster ID     | comment_cnt | 어떤 담론 유형이 더 논쟁을 유발하는가 |
| 구간별 좋아요 분포  | section        | like 수     | 시기별 여론 반응 강도 변화            |
| 구간별 댓글 수 분포 | section        | comment_cnt | 시기별 논쟁 활성도 변화               |
| 토픽별 좋아요 분포  | dominant_topic | like 수     | 잠재 토픽에 따른 공감 반응 차이       |

**이상치 처리**: 좋아요·댓글 수는 상위 1% 이상치가 시각화를 왜곡할 수 있으므로 `flierprops` 로 이상치를 작게 표시하거나 99 percentile로 Y축 상한을 설정한다.

```python
# 예시: 군집별 좋아요 수 박스플롯
sns.boxplot(data=doc_df, x='cluster_id', y='like',
            order=[0, 1, 2], palette='Set2')
plt.ylim(0, doc_df['like'].quantile(0.99))
```

### 6.3 히트맵 + 박스플롯 결합 해석

두 시각화를 함께 사용하면 담론 분석을 두 층위로 볼 수 있다.

```
히트맵  →  "구간 3에서 군집 1(정책 논쟁형)이 급증"
박스플롯 →  "군집 1의 댓글 수 중앙값이 다른 군집의 2배"
            ⟹ 구간 3에서 정책 논쟁 담론이 양적으로 증가했을 뿐 아니라
               여론 상호작용(논쟁성)도 가장 활발했다
```

## 7. LDA vs K-Means 비교 관점

두 분석은 **상호 보완적**으로 해석한다.

| 구분      | K-Means                         | LDA                                      |
| --------- | ------------------------------- | ---------------------------------------- |
| 접근 방식 | Hard Clustering (1문서 = 1군집) | Soft Clustering (1문서 = 복수 토픽 확률) |
| 결과 형태 | 군집 중심 키워드                | 토픽별 단어 분포                         |
| 강점      | 군집 크기·분리도 명확           | 혼합 토픽 문서 처리 유연                 |
| 약점      | 혼합 주제 문서 오분류 가능      | 토픽 수 사전 설정 필요                   |
| 활용      | 구간별 여론 클러스터 유형화     | 잠재 주제 발굴 + 시계열 변화             |

**해석 전략**: K-Means 군집 레이블과 LDA 토픽 레이블이 일치하면 해당 담론이 구조적으로 안정적임을 의미. 불일치 시 LDA의 혼합 토픽 특성을 활용하여 세부 분해.
